Nota sobre el sistema y el código abierto

CoopTrack está pensado como un sistema abierto, legible y auditable. Detrás de cada jugada no hay una magia oculta sino una línea de código institucional que puede revisarse.

La base de esa lectura es el play_code, una cadena con segmentos separados por §. En ella se guarda la información esencial de cada jugada: mazo, usuario, fecha, carta, palo, acción, autorizados, flujo y destinatarios.

El archivo engine/playParser.js es una de las piezas centrales de esa lógica. Su tarea es leer cada play_code, separarlo, normalizarlo y validarlo para que el sistema pueda interpretar qué clase de jugada es, qué relación tiene con otras y si su estructura mínima es correcta.

Dicho de otro modo: una parte importante de la lógica de CoopTrack no depende solo de pantallas, botones o formularios, sino de cómo se escriben y se leen esas líneas del libro de jugadas. Por eso, quien quiera comprender el funcionamiento interno del sistema puede ir a revisar directamente ese archivo y seguir desde allí la semántica de cada carta.
